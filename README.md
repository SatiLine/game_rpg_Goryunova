# Текстовая RPG с AI-персонажами

## Описание проекта

Данный проект представляет собой текстовую RPG-игру с искусственным интеллектом для неигровых персонажей (NPC). Игрок перемещается по локациям средневекового города, общается с NPC, настроение которых определяется ML-моделью на основе игрового контекста. Диалоги генерируются либо с помощью шаблонов, либо с использованием большой языковой модели через Ollama (опционально).

Проект реализован в микросервисной архитектуре: веб-интерфейс на Flask, ML-сервис на FastAPI, база данных PostgreSQL.

---

## Актуальность

Современные компьютерные игры стремятся к созданию живых, реалистичных миров, где NPC обладают собственным характером и динамически реагируют на игрока. Традиционные системы диалогов на основе жёстко заданных сценариев ограничены и не могут обеспечить разнообразие взаимодействий.

Использование ML для предсказания настроения NPC и генерации ответов позволяет:

- сделать игровой мир более непредсказуемым и интересным;
- снизить затраты на ручное написание диалогов;
- персонализировать опыт каждого игрока;
- исследовать применение AI в игровых механиках.

---

## Цель проекта

Разработка многокомпонентной RPG с элементами AI: предсказание настроения NPC на основе игрового контекста и генерация контекстных диалогов.

---

## Используемая ML-модель

### RandomForestClassifier для предсказания настроения

Модель обучается на синтетических данных, где признаками выступают:

- `game_hour` — текущий час в игре (0–23);
- `player_gold` — количество золота у игрока;
- `player_hp` — здоровье игрока;
- `npc_role_id` — закодированная роль NPC (кузнец, трактирщица, торговец, стражник, маг);
- `npc_busy` — занятость NPC (0 или 1).

Выход модели — одно из четырёх настроений: `hostile` (враждебное), `neutral` (нейтральное), `friendly` (дружелюбное), `busy` (занятое).

### Генерация диалогов (опционально)

При наличии доступа к Ollama используется большая языковая модель (например, `qwen2.5:1.5b`) для генерации живых ответов NPC с учётом его личности, памяти и текущего настроения. Если Ollama недоступна, система автоматически переключается на заранее подготовленные шаблоны фраз.

## Задачи проекта

- разработать микросервисную архитектуру (Flask + FastAPI);
- реализовать REST API для предсказания настроения NPC;
- обучить RandomForest модель на синтетических данных;
- подключить PostgreSQL и Alembic для миграций;
- реализовать систему авторизации пользователей;
- сохранять историю диалогов в базе данных;
- реализовать веб-интерфейс с перемещением по локациям;
- внедрить систему состояний и расписания NPC;
- опционально интегрировать Ollama для генерации диалогов;
- настроить логирование и обработку ошибок;
- добавить тесты (pytest);
- контейнеризировать сервисы (Docker Compose);
- использовать современные инструменты: `uv`, `ruff`, `mypy`, `pre-commit`.

---

## Функциональные требования

Система должна обеспечивать:

- регистрацию и авторизацию пользователей;
- отображение игрового мира с локациями и переходами;
- взаимодействие с NPC (диалоги);
- предсказание настроения NPC на основе контекста (здоровье, золото, время, роль);
- генерацию ответа NPC (шаблоны или Ollama);
- сохранение истории всех взаимодействий;
- автоматическое изменение времени суток;
- обновление состояний NPC по расписанию.

---

## Нефункциональные требования

Система должна:

- быстро отвечать на действия игрока;
- быть устойчивой к ошибкам ML-сервиса (fallback на шаблоны);
- обеспечивать безопасность пользовательских данных (хеширование паролей, CSRF-защита);
- поддерживать расширяемость (легко добавлять новые локации, NPC, модели);
- быть контейнеризируемой (Docker);
- поддерживать тестирование (pytest);
- соответствовать стандартам кода (ruff, mypy);
- использовать управление зависимостями через `uv`.

---

## Архитектура системы

### Основные компоненты:

- **Flask (веб-интерфейс)** — обработка пользовательских действий, рендеринг страниц, авторизация, взаимодействие с БД.
- **FastAPI (ML API)** — загрузка модели, предсказание настроения, интеграция с Ollama.
- **PostgreSQL** — хранение пользователей и истории предсказаний.

Связь: Браузер → Flask :5000 → FastAPI :8000 → (опционально) Ollama.

---

## Общая схема работы

```text
Браузер (игрок)
  ↓
Flask (веб-сервер) :5000
  ↓
Авторизация (сессии)
  ↓
Игровая логика (локации, NPC, диалоги)
  ↓
FastAPI :8000 (ML-сервис)
  ↓
PostgreSQL (хранение данных)
```
